# Untrusted-repo copy hardening (tree limits + executable/hook safety)

Deferred from the `configless-install` discussion. Configless install widens an already-open aperture: agntc clones arbitrary third-party repos (`cloneSource`, `git clone --depth 1`) and recursively copies their contents into the project (`copyBareSkill`, `cp(sourceDir, destDir, { recursive: true })`). The configless feature itself folds in the *cheap, boundary-level* guards — path-traversal validation on the source selector/subpath (Vercel's `isSubpathSafe` floor) and symlink-escape rejection during copy. This idea captures the two *deeper* hardening concerns that were explicitly scoped out of that feature, to be discussed together later.

**1. Tree size / file-count limits.** `cloneSource` has no size cap and `copyBareSkill` copies the whole tree unbounded. A pathological or hostile repo could fill the disk or produce an absurd install. Worth defining sensible caps (total bytes, file count, max depth) with a clear error when exceeded, rather than copying blindly. Vercel skips `node_modules/.git/dist/build/__pycache__` during its walk — a similar ignore list plus a ceiling would help.

**2. Executable / hook safety.** A configless "plugin" with a `hooks/` directory installs hook scripts that run on the agent's next invocation — effectively arbitrary code execution, with less scrutiny than a config-bearing install used to imply. This is partly inherent to the plugin asset model (installing a plugin means accepting its hooks), but configless lowers the bar to reach it. Options to discuss: surfacing/【confirming hooks at install time, flagging executable files, or some form of content review before hooks land in `.claude/hooks/`.

Both are framed as a general "validation / untrusted-content" concern, not configless-specific — they apply to agntc's existing copy path regardless. The configless work just makes them more pressing because the trust gate ("repo shipped an `agntc.json`") is gone. Relevant files: `src/git-clone.ts` (`cloneSource`), `src/copy-bare-skill.ts` (`copyBareSkill`). Reference floor: Vercel `skills` — YAML-only frontmatter (no `---js` eval), `isSubpathSafe`, walk ignore-list.
